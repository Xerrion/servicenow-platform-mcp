Polling Implementation:
- Parameters clamped: poll_interval [2-30s], max_poll_duration [10-300s]
- Uses time.monotonic() for elapsed time tracking (monotonic clock)
- Terminal states: "completed", "failure", "error", "cancelled" (normalized to lowercase)
- Timeout handling: Returns polling_timeout status with warning message

Health Computation (atf_test_health):
- Pass rate: pass_count / total_runs (handles division by zero)
- Flaky detection: Counts transitions between pass/fail states
  - Heuristic: flaky if transitions >= 3 OR (transitions >= 2 AND total_runs < 10)
- Trend analysis: Compares first-half vs second-half pass rates
  - improving: diff > +5%, degrading: diff < -5%, stable: otherwise
  - insufficient_data: if total_runs < 4
- Date filtering: sys_created_on >= javascript:gs.daysAgoStart(N)
