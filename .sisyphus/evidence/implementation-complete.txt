## Key Implementation Decisions

### Cloud Runner API Integration
- Defensive response parsing (handles both `{"result": {...}}` and raw JSON)
- 404 detection with plugin installation message
- `_raise_for_status()` called for all responses

### Polling Implementation
- Explicit defaults: max_timeout=300s, interval=5s, configurable
- Hard timeout cap at 300s (cannot be exceeded)
- Uses `time.monotonic()` for elapsed time tracking
- Graceful handling of all terminal states

### Write Gating
- Both execution tools (`atf_run_test`, `atf_run_suite`) check write gate FIRST
- Return error envelope if blocked (before API call)
- Tests verify gate enforcement

### Test Health Analysis
- Pass rate calculation with division-by-zero protection
- Flaky test detection (failed at least once AND passed at least once)
- Trend detection (multiple recent failures)
- Handles empty result sets gracefully
